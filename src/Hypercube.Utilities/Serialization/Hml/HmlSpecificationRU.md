# HML — Hypercube Markup Language
Минималистичный декларативный язык описания данных со строгой типизацией.

Текущая редакция: март 2026

## 1. Идентификаторы

Правила формирования имён полей, свойств, меток:

- начинается с буквы (a–z, A–Z) или символа `_`
- далее допускаются буквы, цифры, `_`
- чувствителен к регистру
- **не может содержать** [зарезервированные символы](#3-зарезервированные-символы-и-последовательности)

Примеры:
- `name`
- `userLevel`
- `_config`
- `HttpStatus2xx`

## 2. Значения и завершение выражений
После `:` идёт значение.  
Завершается значение одним из способов: перенос строки (EOL) или `,`

Примеры:

_Вариант A_:
```hml
health: 100
name: "Sara"
online: true
```

_Вариант B_:
```hml
health: 100,
name: "Sara",
online: true
```

_Вариант C_:
```hml
health: 100, name: "Sara", online: true
```

## 3. Зарезервированные символы и последовательности

Эти символы и последовательности зарезервированы для синтаксиса языка и **нельзя** использовать внутри идентификаторов.

| Категория               | Символы / последовательности                                          | Назначение / примечание                           |
|-------------------------|-----------------------------------------------------------------------|---------------------------------------------------|
| Структурные скобки      | `{ } [ ] ( )`                                                         | начало/конец объектов, списков, словарей, групп   |
| Разделители и терминалы | `: ,`                                                                 | разделитель ключа и значения, элементов коллекций |
| Устаревшие терминалы    | `;`                                                                   | нет                                               |
| Строки                  | `'` `"` `"""`                                                         | одинарные, двойные и тройные кавычки              |
| Типы и перечисления     | `< > $`                                                               | `<Type>`, `$Enum.Value`                           |
| Комментарии             | `//` `/*` `*/`                                                        | однострочные и многострочные комментарии          |
| Операторы (арифметика)  | `+ - * / % **`                                                        | математические операции                           |
| Операторы (побитовые)   | `& ^ ~ << >> >>>`                                                     | побитовые операции                                |
> **Важное замечание**  
> Символ `;` остаётся в списке зарезервированных, но **не поддерживается** как способ завершения значения в новых документах.

## 4. Тип unknown

Тип `unknown` присваивается в следующих случаях:
- значение не удалось однозначно распознать
- после вычисления выражения осталось несколько элементов, разделённых запятой (например, `3/16, -5/16`)
- явно указан пользовательский тип (`<InventoryItem>`, `<Rect2>`, …), но десериализатор его не знает

Примеры:
```
position: 120, 48               → unknown ("120, 48")
offset:   3/16, -5/16           → unknown ("0.1875, -0.3125")
item:     <ItemStack> "dirt#32" → unknown с меткой <ItemStack>
```

## 5. Явное указание типа

Используется для:
- подсказки десериализатору о желаемом типе
- передачи пользовательских типов
- принудительной интерпретации неоднозначных значений

Примеры:
```
position: <Vector2> { x: 32, y: 32 }
color: <Color> 0xffaa33
offset: <Vector2Another> 0.25, -0.375
key: <KeyCode> 'F5'
```
